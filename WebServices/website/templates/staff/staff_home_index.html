Staff's Home

This is the home for the staff users; i.e., for those users with privileges for managing the main resources of the network. Sections below contain a brief description of the tools available for performing the regular maintenance operations that the network, in accordance with its design, may require. In case any kind of problem appears that was not taken into account in the original design, a set of external tools is provided for performing exceptional maintenance operations on the system.

Regular Maintenance Operations

User Management Operations

These operations permit the management of user accounts in accordance to their lifecycle. This lifecycle is graphically defined in the figure on the right, in which a given user account passes through the different states that are permitted within this system.

  1. Users are initially unregistered.
  2. In order to access to the system, they need to create an account in the system. This account creation will appear as a pending to any staff user.
  3. When a user makes a registration of a given email address in the system, the server automatically sends a verification email to the user.
    • The link provided expires after 4 days. If it was not used, the still existing but inactive account will be deleted by the system maintenance periodic task.
  4. After the user confirms its email address, all staff for the site are notified through email and must accept the new user before it gets access to the system:
    • In case the information provided by users cannot be trusted, staff users can delete that request.
    • If the request is repeated several times with the same email address, staff users can block that email address so that subsequent requests will not appear as pending again.
  5. Those who get accepted into the system will receive a notification through email.
  6. Accepted users can be either blocked or deleted, as required.

Exceptional Maintenance Operations

Since exception operations, by definitions, cannot be defined beforehand, this section simply presents the set of tools available for dealing with these issues. These tools can only be accessed within the same network of the server node.

  • Django Admin , an external Django application manager.
  • PhpMy Admin , an external database management tool.
